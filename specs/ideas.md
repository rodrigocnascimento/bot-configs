# Possible future implementations

We could create a skill to the compact the last changes, and get back to work.
The steps go as follow bellow:

1. Enter in the branch of stable development (develop)
2. Compare with the release stable branch (main)
3. Read the package.json version and compare with the last stable version
4. Read the CHANGELOG
5. Read the last 10 commits and understand last changes
6. Create a report file on the specs/ folder with the last changes.
